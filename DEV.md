# DEV

## ZIP file structure

The ZIP downloaded from DeepSeek Platform contains a directory with two CSV files:

```
usage_data_YYYY_M/
├── amount-YYYY-M.csv
└── cost-YYYY-M.csv
```

- The directory name follows the pattern `usage_data_<year>_<month>` (e.g. `usage_data_2026_5`).
- The CSV files are named `amount-YYYY-M.csv` and `cost-YYYY-M.csv` (non-zero-padded month).

## CSV formats

### cost.csv

| Field       | Type   | Description                              |
|-------------|--------|------------------------------------------|
| user_id     | UUID   | DeepSeek account identifier              |
| utc_date    | date   | UTC date of the usage (YYYY-MM-DD)       |
| model       | string | Model name (e.g. `deepseek-v4-pro`)      |
| wallet_type | string | Wallet type (e.g. `Paid`)               |
| cost        | float  | Cost in the given currency               |
| currency    | string | Currency code (e.g. `CNY`)              |

This file aggregates cost per user per model per day. It does **not** include per-API-key granularity.

### amount.csv

| Field        | Type   | Description                                               |
|--------------|--------|-----------------------------------------------------------|
| user_id      | UUID   | DeepSeek account identifier                               |
| utc_date     | date   | UTC date of the usage (YYYY-MM-DD)                        |
| model        | string | Model name                                                |
| api_key_name | string | User-defined name for the API key                         |
| api_key      | string | Masked API key (middle chars replaced with `*`)           |
| type         | string | Row type (see below)                                      |
| price        | float  | Unit price in CNY (empty for `request_count`)             |
| amount       | int    | Quantity (token count or request count)                   |

**`type` values:**

| type                     | Description              |
|--------------------------|--------------------------|
| `output_tokens`          | Tokens generated by API  |
| `input_cache_hit_tokens` | Input tokens served from cache (discounted) |
| `input_cache_miss_tokens`| Input tokens not in cache (full price)     |
| `request_count`          | Number of API requests   |

Note: `price` is empty for `request_count` rows since they are not billed separately.

## Data flow

1. User uploads ZIP → `JSZip` extracts the two CSV files in memory.
2. `parseCSV()` converts CSV text into arrays of row objects.
3. Charts are rendered from the parsed data:
   - **cost.csv** → daily cost stacked bar chart.
   - **amount.csv** → all other charts (token doughnut, daily token lines, per-key cost & token bars, key summary cards).
4. The API key filter operates on `amount.csv` data only, since `cost.csv` lacks the `api_key_name` field.
